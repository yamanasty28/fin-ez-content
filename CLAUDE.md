# fin-ez-content — FIN-EZ 記事配信リポジトリ

このリポジトリの `articles.json` は、金融ニュースアプリ **FIN-EZ** が起動時に取得する記事フィード。
GitHub Pages（`https://yamanasty28.github.io/fin-ez-content/articles.json`）で配信される。
**main に push した瞬間、世界中のアプリユーザーに記事が公開される。** ルール違反は審査リジェクト・法的リスクに直結するため、以下を厳守すること。

---

## アプリのコンセプト

- **「知識ゼロでもわかる・今日のお金の話」** — 小学生・中学生でも理解できる金融ニュースアプリ
- ターゲット：NISAを始めたばかり・経済ニュースに挫折した金融初心者
- 複数の実在ニュースソース（FT, WSJ, Bloomberg, Reuters, 日経, CoinDesk など）を参考に、**要約ではなく完全な独自記事**を生成する
- 「教える」立場であって「勧める」立場ではない（投資助言にならないこと）

---

## 毎日の記事生成手順

**毎朝、最低7本。ニュースが多い日はそれ以上（目安7〜10本、出来事が特に多ければ10本超も可）出す。** 本数は固定しない。「全5ジャンルを最低1本ずつ必ずカバー」した上で、**その日にニュースが多い・人気の高いジャンルに記事を上乗せ**して合計を増やす。

**本数の決め方：**

- まず5ジャンル（`stocks` / `crypto` / `fx` / `economy` / `money`）に**最低1本ずつ＝土台5本**。
- そこに、その日**出来事が多い／影響が大きいジャンルへ追加で2本以上**を上乗せして**合計7本以上**にする。株式・仮想通貨・経済はニュースが多くなりやすく厚めになりがち。為替・身近なお金は手薄になりやすいが、最低1本は必ず確保する。
- ビッグニュースが集中した日は1ジャンルで3本以上書いてもよい。**質を最優先し、無理な水増しはしない**（薄い記事を本数合わせで足さない）。ただし最低ラインの7本は守る。

各記事について、次を行う：

1. **Web検索でそのジャンルの「今日」の実際のニュースを調べる**（最低2つの独立ソースで事実確認）
2. 初心者に最も影響・学びが大きいトピックを選ぶ。1ジャンルで複数書くときは別々の出来事／切り口にし、内容が重複しないようにする
3. **そのジャンルに当日めぼしいニュースが乏しい場合**は、捏造せず、時事に絡めた**普遍的なお金の知識記事**（そのジャンルの基礎解説・仕組み解説）を書く。「ニュースが無いから書かない」ではなく「ニュースが弱い日は学びの記事」にして、必ず最低1本は埋める
4. 下記ルールに従ってインフォグラフィック記事を書く。id は `daily-YYYYMMDD-<genre>`（例 `daily-20260613-stocks`）。**同じ日に同ジャンルで2本目以降を書くときは末尾に連番を付ける**（`daily-20260613-stocks-2`, `-3` …）。id は重複させない
5. **その記事で説明を付けた専門用語を洗い出し、`bundled-terms.json`・`terms` のどちらにも無い語は必ず `terms` に追加する（下の「📖 辞書ルール」参照・スキップ厳禁）**

全部そろえたら：

6. **無料はちょうど3本、残りは全部プレミアム限定（`isPremium: true`）にする。** 無料3本はその日の最重要ニュース。下の「💎 プレミアム限定ルール」参照
7. `articles.json` の `articles` 配列の**先頭**に当日分をまとめて追加（重要度の高い順）し、`updatedAt` を更新する
8. 記事数が **200本** を超えたら古いものから削除する（`terms` は削除せず蓄積）
9. **検証**：`python3 -c "import json; json.load(open('articles.json'))"` でJSONが壊れていないこと、NGワードがどの記事にも含まれないこと、**当日分が7本以上**で**無料（`isPremium:false`）がちょうど3本・残り全部 `isPremium:true`** であること、全5ジャンルが最低1本あること、id に重複が無いこと、**当日記事すべてに `region`（domestic/overseas/basics のいずれか）が付いていること・暗号資産は必ず overseas であること（「🌍 地域ルール」参照）**、**当日記事で説明した専門用語がすべて bundled-terms.json か terms に存在すること（辞書ゲート・「📖 辞書ルール」参照）**を確認
10. main にコミット & push（コミットメッセージ例：`Add daily articles 2026-06-13 (8 articles, 3 free)`）

> 💡 最低ラインは「全5ジャンルを1本以上カバー・合計7本・無料ちょうど3本」。出来事が多い日はニュースの多いジャンルを厚くして増やす。質を落としてまで本数を増やさない。

## 🌍 地域ルール（`region` フィールド・毎記事必須）

ジャンル（テーマ）とは別に、**各記事に地域軸 `region` を必ず1つ付ける**。アプリの「国内／海外／学びの基本」フィルタに使う。値は3つ：

| `region` | 表示ラベル | 何を入れるか |
|---|---|---|
| `domestic` | 国内 | 日本固有の出来事・制度・数値。日銀・日経平均・消費税・NISA・iDeCo・フラット35・日本のCPI・為替介入・**日米比較（日本の読者目線の記事）** |
| `overseas` | 海外 | 米国/世界市場のニュースと、**暗号資産の記事はすべてここ**（値動きニュースも仕組み解説も）。FRB・アメリカ・S&P/ダウ/ナスダック・海外IPO・ビットコイン/イーサリアム/ステーブルコイン等 |
| `basics` | 学びの基本 | 場所に依らない普遍的な仕組み解説。複利・単利・分散投資・PER・配当・GDPとは・インフレとは・損益分岐点・機会費用・固定費/変動費・先取り貯金 など |

**判定のコツ：**「特定の国の"今日の出来事"か？」→ 日本なら `domestic`、米国・世界なら `overseas`。「時事に依らない"仕組みの学び"か？」→ `basics`。**暗号資産（crypto ジャンル）は内容を問わず必ず `overseas`**（海外発の資産のため。過去分もこのルールで統一済み）。日米を比べる記事は日本の家計・投資家目線が主題なので `domestic`。

> 💡 `region` は5ジャンルとは独立。1日分の中で地域が偏っても構わない（無料3本の選び方・ジャンル網羅ルールが優先）。ただし全記事に必ず1つ付けること。

## 🧭 「やってみよう」シリーズ（実践・手順の記事）ルール

ニュース解説（毎日の当日分）とは別に、**「実際に始める人向けの"やり方・読み方"」を教える常設シリーズ**。金融リテラシーの実践パート。**id は `howto-<スラッグ>`**（例 `howto-candlestick`）で、日付に依らない**常設記事（エバーグリーン）**。当日分ではないので「無料ちょうど3本」ルールの対象外＝入門の看板になる回は `isPremium:false`（無料）にしてよい。`genre`・`region`・辞書ルール・NGワード・初心者トーンは通常記事と完全に同じ。`region` は原則 `basics`（口座・税制など日本固有の手続きは `domestic`、暗号資産は `overseas`）。

**扱うテーマ（手順・道具の使い方に徹する）：** 証券口座やNISA口座の開き方の流れ、注文画面の使い方（指値・成行の出し方）、チャートの読み方（ローソク足・移動平均線が"何を表す線・記録か"）、積立の設定手順、デモ口座での練習、暗号資産の口座開設と2段階認証など。

**🚨 このシリーズ特有の禁止（通常のNGワードに追加）：**

| 禁止 | 理由・言い換え |
|---|---|
| 特定の証券会社／取引所／銘柄を名指しで推奨・ランク付け | 一般名詞（「証券会社」「取引所」）で書く。比較して「どこが一番」は書かない |
| チャートの形・指標を売買シグナルとして断定（「〇〇が出たら買い」「ゴールデンクロスは買い」） | 「〜を表す線です」「値動きを読む練習の道具」まで。売買の合図とは書かない |
| 「これで儲かる」「勝てる」「初心者でも稼げる」 | 手順・仕組みの説明に徹する。成果を約束しない |
| いつ買う／売るというタイミング指示 | 「自分で判断する」前提を必ず添える |

**必須：** 各記事の末尾に `note`（label 例「ご注意」）で**投資助言でない旨の免責**を入れる。暗号資産テーマなら価格変動・元本割れリスクの一文も併記。「これは仕組み・手順の解説で、特定の売買を勧めるものではありません。投資の判断はご自身の責任で」の趣旨。

**運用：** 毎日の7本の中にこのシリーズを **1〜2本** まぜてよい（当日分の daily 記事とは別枠で追加してもよい）。急がず、質を最優先に増やす。

## 💎 プレミアム限定ルール（収益化の要）

無料ユーザーは1日3本まで読める。全部を無料にすると「プレミアムに入る意味」が無くなるため、**毎日の当日分のうち無料はちょうど3本だけ・残りは全部プレミアム限定（`isPremium: true`）にする。** 本数を増やすほど無料3本は変えず、増えた分はすべてプレミアム側に積む。

- **無料にする3本**：その日の最重要ニュース。誰でも読める"看板記事"。全ユーザーがちゃんと情報を得られるようにする。なるべく違うジャンルから選ぶ
- **プレミアム限定（残り全部）**：より深掘りした記事・応用的な学びの記事。「核心ニュースは無料、もっと知りたい人はプレミアム」という形にする（重要ニュース自体を有料の壁で隠さない）
- 限定記事も品質・NGワード・初心者トーンのルールは完全に同じ。手抜き記事をプレミアムにしない
- アプリ側は `isPremium:true` の記事を未加入ユーザーにはロック表示（タイトル＋要約＋「プレミアムを見る」ボタン）する

> ⚠️ 「無料ちょうど3本」は審査・景表法（無料本数の表記一致）にも関わる固定値。本数の上限（7本以上）は変えてよいが、**無料はいつでも3本ちょうど**にすること。

## 🚨 ハルシネーション防止（最重要）

1. 検索で確認した数値・データ以外の価格・数字を推測・捏造しない
2. 未来の価格予測・値上がり予測を絶対に出力しない
3. 事実と推測を明確に分ける（「〜と言われています」「〜という見方があります」）
4. 架空のニュース・イベント・日付を生成しない
5. 他社メディアのスタイル模倣はしない。常に「FIN-EZ独自の初心者向けスタイル」
6. 事実確認が取れるニュースが見つからない日は、時事に絡めた**普遍的なお金の知識記事**（`money` ジャンル）を書く（捏造するくらいならニュースに触れない）

## 文章ルール

- **本文を文字の壁にしない。** 読者が「読む」前に「眺めて分かる」紙面にする。図解系ブロック（flow / chart / numbers / compare）を1記事に最低3つ使う
- **小学生・中学生が読んでわかる文章。** 「これ知ってる？」と思う用語は必ず説明する。説明しすぎは存在しない
- 一文は60文字以内。英語はカタカナ＋（説明）をセットで
- 略語・指標（PER・ETF・GDP…）は初出で必ず「正式名称＋ひとことで何か」をセットで書く
- ドル・外貨表記には必ず円換算を添える：「約10ドル（約1,500円分）」
- 数字には必ず比較・変化を添える：「5.19%（1年前は4.5%。19年ぶりの高水準）」
- 日常生活への例え（イメージするなら…）を必ず入れる

### 専門用語の言い換え（必ず使う）

| 用語 | 言い換え |
|---|---|
| 日経平均 | 「日本を代表する225社の株価の平均」 |
| ダウ平均 | 「アメリカを代表する30社の株価の平均」 |
| S&P 500 | 「アメリカの主要500社の株価をまとめた指数」 |
| 決算 | 「会社の3ヶ月に1回の成績発表。学校の通知表みたいなもの」 |
| 時価総額 | 「その会社全体の"値段"。株価×株数で計算する」 |
| 金利 | 「お金を借りたり預けたりするときの"手数料率"」 |
| 利下げ／利上げ | 「借りるお金の手数料を安く／高くすること」 |
| 国債 | 「国が発行する借用書。満期になると利子をつけて返してくれる」 |
| FRB | 「アメリカ版・日本銀行。お金の量と金利を調整する人たち」 |
| 日銀 | 「日本のお金の量と金利を管理する国の機関」 |
| インフレ | 「コンビニのおにぎりが去年より高くなってるあの現象」 |

## 📖 辞書ルール（記事と同じくらい重要・スキップ厳禁）

**記事の中で「説明が必要」と判断して噛み砕いた専門用語は、1語残らず辞書に載っていること。** 記事で `(○○とは…)` のように説明を付けた用語・略語・指標（例：日経平均、FRB、PER、ETF、利回り、リスクオフ、半減期 など）は、**必ず `bundled-terms.json` か `articles.json` の `terms` のどちらかに存在する**状態にする。無ければ `terms` に追加する。**「記事は出したが辞書は増えていない」は不合格。**

> ⚠️ これまで記事だけ増えて辞書が増えない日が続いた。**毎回必ず最低1語は見直す**つもりで洗い出すこと（同梱済みばかりで本当に新規ゼロの日もあり得るが、その場合は「全用語が既存だと確認した」こと自体が手順の完了条件）。

手順（毎記事・毎日必ず）：

1. 記事を書き終えたら、**その記事で説明を付けた専門用語をすべて箇条書きで洗い出す**（説明を付けた＝初心者に必要な用語、の印）
2. 洗い出した各語について `bundled-terms.json`（アプリ同梱済み102語の一覧）と `articles.json` の `terms` を**両方**検索し、**どちらにも無い語だけ**を `terms` に追加する（重複追加は禁止）
3. id は `term-daily-<スラッグ>` 形式（例：`term-daily-hangenki`）。`relatedTermIds` には同梱用語の id（bundled-terms.json 参照）も使ってよい
4. 定義も記事と同じトーン：小学生・中学生でもわかる言葉、`simple` は絵文字つきの一言例え、`exampleSentence` は生活が浮かぶ例文。NGワード・投資推奨表現は辞書でも禁止
5. **検証ゲート**：push 前に「当日記事で説明した用語 → すべて bundled か terms にある」ことを1語ずつ確認する。1語でも欠けていたら追加してから push する

```ts
interface Term {
  id: string;                  // 'term-daily-<スラッグ>'
  term: string;                // 例 '半減期'
  reading: string;             // ひらがな。英略語などは ''
  simple?: string;             // 一言でわかる例え（絵文字つき）例 '持っているだけでもらえるお小遣い💰'
  category: 'stocks' | 'crypto' | 'fx' | 'economy' | 'money' | 'general';
  shortDefinition: string;     // 1〜2文の短い定義
  fullDefinition: string;      // 3〜4文の詳しい説明（初心者向け）
  exampleSentence: string;     // 「」つきの生活が浮かぶ例文
  relatedTermIds: string[];    // 関連用語のid
  difficulty: 1 | 2 | 3;       // 1=超初心者 2=初心者 3=中級
  isPremium: boolean;          // 原則 false
}
```

## 記事のデータ形式

`articles.json` 全体：

```json
{
  "updatedAt": "2026-06-12T06:00:00+09:00",
  "articles": [ /* Article を新しい順に */ ],
  "terms": [ /* 追加した辞書用語（Term）。記事と違い削除せず蓄積する */ ]
}
```

### Article（全フィールド必須。`infographic` で本文を構成し `body` は空文字）

```ts
interface Article {
  id: string;              // 'daily-YYYYMMDD-<genre>' 形式（例 daily-20260613-stocks）
  genre: 'stocks' | 'crypto' | 'fx' | 'economy' | 'money';
  region: 'domestic' | 'overseas' | 'basics'; // 地域軸（下記「🌍 地域ルール」参照・必須）
  title: string;           // 「！」「？」を使った初心者が読みたくなる見出し
  summary: string;         // 2〜3文。アプリの一覧と記事冒頭に表示される
  body: '';                // インフォグラフィック記事では空文字固定
  infographic: IGBlock[];  // 本体（下記）
  readingTimeMinutes: number; // 3〜5
  xpReward: number;        // 12〜20（読了時間に比例）
  publishedAt: string;     // '2026-06-12T06:00:00+09:00'（日本時間）
  isPremium: boolean;      // 原則 false
  tags: string[];          // 例 ['日経平均', 'AI', '半導体']
}
```

### IGBlock（インフォグラフィックのブロック）

```ts
type FinPose = 'fin1'|'fin2'|'fin3'|'fin4'|'fin5'|'fin6'|'fin7'|'fin8'; // マスコット犬「Fin」
type IGTone = 'b'|'g'|'p'|'r'|'o'|'y'; // フローのアイコン背景色

type IGBlock =
  | { kind: 'card'; num: number; title: string; body: string[];
      char?: { pose: FinPose; bubble: string };          // Finの吹き出し（イメージするなら…）
      pill?: { text: string; emphasis?: string } }       // 重要数字の黄色強調
  | { kind: 'section'; text: string }                    // 青バー見出し
  | { kind: 'flow'; title: string;
      steps: { emoji: string; tone: IGTone; label: string; sub: string }[] } // subに「なぜ？」必須
  | { kind: 'qa'; items: { q: string; a: string }[] }    // 初心者Q&A 2〜3問
  | { kind: 'compare'; title?: string;
      items: { emoji?: string; label: string; value: string; note?: string }[] }
  | { kind: 'numbers'; title?: string;

  | { kind: 'points'; title?: string; items: string[] }  // ✅ 30秒でわかる要約（先頭に必須）
```

> ⚠️ `points` / `life` / `note` ブロックはアプリの **次期ビルド（Build 4 以降）** で描画される。それ以前の審査中ビルド（Build 3）では `infographic` 機能自体が未搭載のため影響なし。

### 標準構成（この並び）

1. `points`（✅「30秒でわかる要約」3行。summary の繰り返しではなく、数字入りで具体的に）
2. `card`（num:1「まずは結論！」＋ Fin の吹き出しで日常への例え ＋ pill で重要数字）
3. `card`（num:2「なぜ〜なの？（原因）」）
4. `section`「どうつながる？（カンタン図解）」＋ `flow`（各ステップの sub に「なぜ？」）
5. `section`「初心者向け解説」＋ `qa`（2〜3問。「これからどうなるの？」には「未来は誰にもわかりません」と答える）
6. `section`「数字で見る今回の動き」＋ `chart` または `numbers`（検索で確認した実数値のみ。確認できない数値はブロックごと省略）
7. `life`（🏠「生活でこう役立つ」2〜3個。**最重要・毎記事必須**。下記参照）
8. `take`（今日の学び）
9. `quiz`（3択クイズ。**`correctIndex` に正解の選択肢番号〈0始まり〉を必ず入れる**。ユーザーが選んで正解するとXPがもらえる。`answer` は解説文）
10. `note`（👪「おうちの人へ」任意。子どもと話すきっかけになる一言など）

### 🏠 「生活でこう役立つ」の書き方（毎記事必須）

ニュースを「読者自身の生活」につなげる具体例を書く。これがFIN-EZの価値の核。

- ✅ 良い例：「利上げのニュースが出たら、住宅ローンを変動金利で借りている人は返済額が変わらないか確認するタイミング」
- ✅ 良い例：「円安が進むと輸入品（小麦・ガソリン）が値上がりしやすい。スーパーのパンの値段にも数ヶ月後に影響することがある」
- ❌ 悪い例：「投資のチャンスです」（推奨NG）・「経済に詳しくなれます」（抽象的すぎ）
- 行動を促すときも「確認する」「知っておく」「比べてみる」までにとどめ、売買は促さない

### 禁止事項

- 実画像・報道写真・実在企業ロゴは使用禁止（著作権）。グラフはchartブロック、キャラはFinのみ
- ページ側がタイトル・要約を表示するため、infographic内でタイトルや要約を繰り返さない
